# Vulkan 파이프라인 완벽 가이드: 래스터화와 레이 트레이싱

이 문서는 Vulkan의 두 가지 주요 파이프라인, 즉 **그래픽스(래스터화) 파이프라인**과 **레이 트레이싱 파이프라인**의 전체 흐름과 각 단계의 역할을 상세히 설명합니다.

---

### 파이프라인이란 무엇인가? (공장 조립 라인)

그래픽스 파이프라인은 3D 모델 데이터(원재료)를 입력받아 최종적으로 화면에 표시될 2D 이미지(완제품)를 만들어내는 **'공장 조립 라인'**에 비유할 수 있습니다. 각 단계는 특정 작업을 수행하며, 이 단계들이 순서대로 연결되어 하나의 파이프라인을 이룹니다.

Vulkan의 가장 큰 특징은 이 '조립 라인'의 설계를 개발자가 매우 상세하게 직접 제어해야 한다는 것입니다. 이렇게 한 번 최적화된 라인을 구축하면, 렌더링 중에는 설정을 바꾸지 않고 대량의 데이터를 매우 빠르게 처리할 수 있습니다.

---

### 1. 그래픽스 파이프라인 (Graphics/Rasterization Pipeline)

가장 전통적이고 널리 사용되는 파이프라인으로, 3D 모델을 2D 픽셀의 집합으로 변환(래스터화)하는 방식입니다.

#### **전체 흐름**

`정점 데이터 입력` -> `1차 가공 (정점 셰이더)` -> `(선택) 추가 가공` -> `도트화 (래스터화)` -> `채색 (프래그먼트 셰이더)` -> `최종 조립 (출력 병합)` -> `완제품 (픽셀)`

#### **파이프라인의 각 단계 (공정)**

1.  **입력 조립기 (Input Assembler)**
    - **역할**: 버텍스 버퍼와 인덱스 버퍼에서 정점 데이터를 읽어와 파이프라인에 공급합니다.
    - **비유**: **'원재료 투입 공정'**. 로봇을 조립하기 위해 필요한 모든 부품(정점)과 조립 순서도(인덱스)를 컨베이어 벨트에 올립니다.

2.  **정점 셰이더 (Vertex Shader) - 프로그래밍 가능**
    - **역할**: 각 정점(Vertex)의 위치를 3D 월드 공간에서 2D 화면 공간으로 변환하는 등, 정점 단위의 연산을 수행합니다.
    - **비유**: **'1차 가공 공정'**. 컨베이어 벨트 위의 각 부품을 최종 조립될 위치에 맞게 옮기고 변형시킵니다. (예: 모델의 로컬 좌표 -> 월드 좌표 -> 카메라 좌표 -> 화면 좌표)

3.  **테셀레이션 & 지오메트리 셰이더 (Tessellation & Geometry Shaders) - (선택) 프로그래밍 가능**
    - **역할**:
        - **테셀레이션**: 평평한 표면을 더 많은 삼각형으로 쪼개어 디테일을 높입니다. (예: 평평한 지형에 울퉁불퉁한 효과 추가)
        - **지오메트리**: 점, 선, 삼각형 같은 기본 도형(Primitive) 단위로 작동하며, 이를 변형하거나 새로운 도형을 생성할 수 있습니다. (예: 정점 하나를 폭발시켜 여러 파편 생성)
    - **비유**: **'특수 가공 공정'**. 밋밋한 부품 표면을 더 정교하게 다듬거나(테셀레이션), 부품 하나를 여러 개로 복제(지오메트리)하는 추가 작업입니다.

4.  **래스터화 (Rasterization) - 고정 기능**
    - **역할**: 이전 단계에서 만들어진 벡터 형태의 도형(삼각형)을 화면에 채워질 픽셀들의 집합, 즉 프래그먼트(Fragment)로 변환합니다.
    - **비유**: **'도트화(Dot-ification) 공정'**. 삼각형 모양의 설계도를 보고, 그 안을 채울 수 있는 모든 점(픽셀)들의 위치를 계산해냅니다. 아직 색은 칠해지지 않은 상태입니다.

5.  **프래그먼트 셰이더 (Fragment Shader) - 프로그래밍 가능**
    - **역할**: 래스터화 단계에서 생성된 각 프래그먼트(잠재적 픽셀)의 최종 색상을 결정합니다. 텍스처에서 색상을 가져오고, 조명 계산을 수행하는 등 가장 연산량이 많은 단계 중 하나입니다.
    - **비유**: **'채색 공정'**. 도트화된 각 점에 어떤 색을 칠할지 결정합니다. 텍스처라는 스티커를 붙이고, 조명의 밝기와 그림자를 계산하여 최종 색을 만들어냅니다.

6.  **출력 병합 (Color Blending) - 고정 기능**
    - **역할**: 프래그먼트 셰이더가 계산한 색상을 프레임버퍼(최종 이미지)에 이미 그려져 있는 색상과 어떻게 혼합할지 결정합니다. (예: 투명 효과) 또한, 뎁스 테스트, 스텐실 테스트 등을 수행합니다.
    - **비유**: **'최종 조립 및 검수 공정'**. 새로 칠한 부품(프래그먼트)을 기존의 반제품(프레임버퍼)과 합칩니다. 만약 투명 부품이라면 아래의 색과 섞고, 불투명하다면 그냥 덮어씁니다. 또한, 이미 더 가까운 부품이 있다면 폐기합니다(뎁스 테스트).

### 2. 레이 트레이싱 파이프라인 (Ray Tracing Pipeline)

빛의 경로를 추적하여 이미지를 생성하는 방식으로, 래스터화와는 근본적으로 다릅니다.

#### **전체 흐름: 탐정 사무소의 사건 처리**

- **비유**: 화면의 각 픽셀은 하나의 '사건 파일'입니다. **'탐정 사무소(GPU)'**는 각 사건을 해결하기 위해 **'탐정(Ray)'**을 현장(3D 씬)으로 파견합니다.

#### **파이프라인의 주요 역할 (셰이더)**

1.  **광선 생성 셰이더 (Ray Generation Shader) - '사건 담당관'**
    - **역할**: 파이프라인의 시작점. 각 사건 파일(픽셀)에 대해 탐정을 파견합니다. 탐정의 출발점(카메라 위치)과 바라볼 방향을 지정하고, `traceRayEXT()` 함수를 호출하여 "출동!" 명령을 내립니다.
    - **임무**: 모든 조사가 끝난 후, 탐정들로부터 최종 보고(색상 값)를 받아 사건 파일(픽셀)을 종결합니다.

2.  **교차 셰이더 (Intersection / Any-Hit / Closest-Hit) - '현장 요원'**
    - **역할**: 파견된 탐정이 현장에서 단서(물체)를 발견했을 때 활동합니다.
    - **Closest-Hit Shader ('결정적 증거 분석가')**: 탐정이 발견한 여러 단서 중 **가장 결정적인 단서(가장 가까운 물체)**에 대해 상세 분석을 시작합니다. "이 벽은 벽돌 재질이고, 저쪽에서 오는 빛을 받아 이 부분은 밝게 빛난다" 와 같이 최종 색상을 결정하는 핵심적인 셰이딩 작업을 수행합니다.
    - **Any-Hit Shader ('투명 단서 분석가')**: (선택 사항) 반투명한 유리창처럼, 광선이 멈추지 않고 통과해야 할 때 사용됩니다. "유리창을 발견했지만, 통과해서 계속 조사하라" 와 같은 명령을 내릴 수 있습니다.

3.  **누락 셰이더 (Miss Shader) - '미제 사건 보고관'**
    - **역할**: 파견된 탐정이 아무런 단서도 찾지 못하고 허공으로 사라졌을 때 활동합니다.
    - **임무**: "현장에 아무것도 없음. 하늘만 보임." 과 같이, 기본 배경색(예: 하늘색)을 보고하여 사건을 종결합니다.

#### **핵심 보조 시스템**

- **가속 구조 (AS)**: '스마트 주소록'. 탐정이 수많은 물체들 사이에서 단서를 빠르게 찾을 수 있도록 돕는 핵심 시스템입니다.
- **셰이더 바인딩 테이블 (SBT)**: '상황별 대응 매뉴얼'. "단서를 찾았을 때 -> 분석가(Closest-Hit) 호출", "단서를 못 찾았을 때 -> 보고관(Miss) 호출" 과 같이, 각 상황에 맞는 셰이더(전문가)를 연결해주는 주소록입니다.

---

### 3. 두 파이프라인의 비교 요약

| 항목 | 그래픽스 (래스터화) 파이프라인 | 레이 트레이싱 파이프라인 |
| :--- | :--- | :--- |
| **기본 원리** | 3D 모델 -> 2D 픽셀로 **투영 및 색칠** | 2D 픽셀 -> 3D 씬으로 **광선 추적** |
| **비유** | 공장 조립 라인 (화가) | 탐정 사무소 (사진가) |
| **강점** | **속도**. 대부분의 실시간 렌더링에 사용. | **사실감**. 반사, 굴절, 부드러운 그림자 표현에 탁월. |
| **약점** | 사실적인 광학 효과를 '흉내'내기 어려움. | 계산 비용이 매우 높음. |
| **핵심 단계** | 정점 셰이더 -> 래스터화 -> 프래그먼트 셰이더 | 광선 생성 -> 교차 테스트 -> Hit/Miss 셰이더 |
| **주요 데이터** | 정점/인덱스 버퍼 | 가속 구조 (AS), SBT |

현대의 고성능 애플리케이션(특히 게임)은 두 파이프라인을 혼합하여 사용하는 **하이브리드 렌더링(Hybrid Rendering)**을 통해 속도와 사실감을 모두 잡으려는 경향이 있습니다. 예를 들어, 기본적인 장면은 래스터화로 빠르게 그린 뒤, 반사나 그림자처럼 사실감이 중요한 부분만 레이 트레이싱으로 처리하는 방식입니다.
